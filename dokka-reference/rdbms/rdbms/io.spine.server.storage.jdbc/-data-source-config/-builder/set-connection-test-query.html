setConnectionTestQuery

public DataSourceConfig.Builder setConnectionTestQuery(String connectionTestQuery)

If your driver supports JDBC4 it is strongly recommended not setting this property. It is for "legacy" databases that do not support the JDBC4 isValid API.

It is the query that will be executed just before a connection is given to you from the pool to validate that the connection to the database is still alive.

Again, try running the pool without this property, an error will be logged if your driver is not JDBC4 compliant.

Default: none
